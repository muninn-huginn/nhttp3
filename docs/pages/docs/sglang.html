SGLang / LLM Serving

Stream LLM inference tokens over HTTP/3. No head-of-line blocking means faster token delivery to clients.

Why HTTP/3 for LLM Serving

ProblemHTTP/2HTTP/3 (nhttp3)
Token streamingSSE over shared TCP — one slow stream blocks allEach stream independent — no HOL blocking
Multiple clientsTCP connection per clientMultiplexed QUIC streams
Mobile inferenceConnection drops on network changeConnection migrates seamlessly
High latencyTCP + TLS = 2 RTTQUIC = 1 RTT

OpenAI-Compatible Server

import nhttp3, json, asyncio

class LLMServer:
    async def __call__(self, scope, receive, send):
        if scope["path"] == "/v1/chat/completions":
            body = await self.read_body(receive)
            request = json.loads(body)

            if request.get("stream"):
                await self.stream_tokens(send, request)
            else:
                await self.complete(send, request)

    async def stream_tokens(self, send, request):
        await send({"type": "http.response.start",
                    "status": 200,
                    "headers": [[b"content-type", b"text/event-stream"]]})

        for token in self.generate(request):
            chunk = {"choices": [{"delta": {"content": token}}]}
            data = f"data: {json.dumps(chunk)}\n\n"
            await send({"type": "http.response.body",
                        "body": data.encode(), "more_body": True})
            await asyncio.sleep(0.02)

nhttp3.serve(LLMServer(), port=4433,
             certfile="cert.pem", keyfile="key.pem")

curl --http3 https://localhost:4433/v1/chat/completions \
  -X POST -H "Content-Type: application/json" \
  -d '{"model":"llama3","messages":[{"role":"user","content":"Hello"}],"stream":true}' \
